# Ramon Roca Pinilla - Portfolio

Academic portfolio showcasing research in protein engineering, antimicrobial resistance, and biotechnology innovation.

## 🚀 Quick Start

```bash
# Install dependencies
npm install

# Start development server
npm run dev

# Build for production
npm run build
```

**New to the project?** Check out our comprehensive [**Getting Started Guide**](./GETTING_STARTED.md) for detailed setup instructions and development environment configuration.

## 🎨 Design System & Storybook

This portfolio includes a comprehensive design system documented in Storybook:

```bash
# Start Storybook development
npm run storybook

# Build Storybook
npm run build-storybook
```

**Live Documentation**: `http://localhost:6006` (when running locally)

## ✨ Key Features

- **Academic Focus**: Optimized for research presentation
- **Dark Theme**: Professional reading experience
- **Accessibility**: WCAG 2.1 AA compliant

## 🛠️ Development

### Prerequisites

- Node.js 18+
- npm or yarn

### Available Scripts

- `npm run dev` - Start development server
- `npm run build` - Build for production
- `npm run preview` - Preview production build
- `npm run preflight` - Run all quality checks
- `npm run storybook` - Start Storybook
- `npm run test:content` - Validate content integrity

### Quality Assurance

Run preflight checks before committing:

```bash
npm run preflight
```

This runs content validation, type checking, and build verification.

## 🔧 Technical Stack

- **Framework**: [Astro.js](https://astro.build/) - Static site generation
- **Components**: React with TypeScript
- **Styling**: Tailwind CSS with custom design tokens
- **Visualizations**: D3.js for interactive graphics
- **Testing**: Vitest with browser testing
- **Quality**: ESLint, Prettier, Husky pre-commit hooks

## 🤝 Contributing

We welcome contributions to this academic portfolio project! Here's how to get started:

### Quick Start for Contributors

1. **Read the guides**: Start with our [Getting Started Guide](./GETTING_STARTED.md)
2. **Fork the repository**: Create your own fork on GitHub
3. **Set up development environment**: Follow the setup instructions
4. **Create a feature branch**: `git checkout -b feature/your-feature-name`
5. **Make your changes**: Follow our [code standards](./docs/development/code-standards.md)
6. **Run quality checks**: `npm run preflight` to ensure all checks pass
7. **Submit a pull request**: Use our PR template and guidelines

### Contribution Guidelines

- **Read our [Contributing Guidelines](./CONTRIBUTING.md)** for detailed processes and standards
- **Follow our [Development Workflow](./docs/development/workflow-guide.md)** for best practices
- **Maintain code quality** using our [Code Standards](./docs/development/code-standards.md)
- **Test thoroughly** across different browsers and devices
- **Respect academic integrity** - don't modify biographical or research content

### Types of Contributions Welcome

- 🐛 Bug fixes and performance improvements
- ✨ New features and enhancements (discuss first in issues)
- ♿ Accessibility improvements
- 📚 Documentation improvements
- 🎨 Design system enhancements
- 🌍 Internationalization support

For questions, create an issue or check our [documentation hub](./docs/README.md).
